# 海洋生物知识库 Project Brief

## 1. 项目背景

海洋生物资源丰富多样，但相关知识分散于各类书籍、科研论文及网络资源中，缺乏一个集中化、结构化、易获取的知识平台。本项目旨在构建一个在线海洋生物知识库，支持知识的浏览、检索、阅读、互动和管理。

**项目性质**：大三软件工程实训项目，4 人团队，25 天开发周期。

**关键业务结果**：
- 用户可浏览、搜索、阅读海洋生物相关电子书和文档
- 管理员可对知识内容进行完整的增删改查管理
- 系统具备基本的安全防护和数据统计能力

## 2. 产品定位

**系统是什么**：
- 一个基于 Web 的海洋生物知识浏览与管理平台
- 以电子书 → 文档的两级结构组织知识内容
- 支持全文检索、富文本阅读、点赞互动、数据统计

**系统不是什么**：
- 不是社交平台，不支持用户间私信、关注等功能
- 不是在线编辑协作工具，文档由管理员单方面维护
- 不是移动端 App，仅支持 Web 端
- 不是电商平台，无付费、会员等功能

**最重要的运行原则**：
- 知识内容的准确性和可读性优先
- 管理操作简单直观，降低内容维护成本
- 安全防护到位，防止常见 Web 攻击

## 3. 目标用户

| 用户类型 | 目标 | 典型场景 |
|---------|------|---------|
| 普通用户（未登录） | 快速获取海洋生物知识 | 浏览电子书列表、阅读文档、搜索感兴趣的内容 |
| 注册用户（已登录） | 参与互动，表达喜好 | 登录后点赞文档、接收点赞通知 |
| 管理员 | 维护知识库内容 | 管理电子书、分类、文档、用户，查看统计数据 |

## 4. 角色和职责

### 普通用户（未登录）
- **可以**：浏览首页、查看电子书列表、阅读文档内容、搜索
- **不能**：点赞、接收通知、访问管理后台
- **默认可见范围**：所有已发布的电子书和文档

### 注册用户（已登录）
- **可以**：普通用户的所有功能 + 点赞文档、接收 WebSocket 通知
- **不能**：管理电子书/文档/分类/用户、访问管理后台
- **默认可见范围**：同普通用户

### 管理员
- **可以**：所有功能 + 后台管理（电子书/分类/文档/用户的增删改查）、查看统计
- **不能**：无限制（拥有系统全部管理权限）
- **特殊权限**：用户管理（新增/编辑/删除/重置密码）

### 系统（自动）
- **定时任务**：每分钟生成电子书数据快照
- **自动处理**：点赞通知通过消息队列异步推送

## 5. 核心流程

### 流程 1：知识浏览（普通用户）
```
首页 → 查看统计概览 + 电子书列表 → 点击电子书 → 进入文档阅读页
→ 左侧目录树选择文档 → 右侧显示富文本内容 → 阅读数 +1
```

### 流程 2：知识搜索（普通用户）
```
首页搜索框输入关键词 → 全文检索（ElasticSearch）→ 搜索结果列表
→ 点击结果 → 进入文档阅读页
```

### 流程 3：文档互动（注册用户）
```
阅读文档 → 点击点赞按钮 → 后端校验（Redis 防重：同 IP 同文档每天一次）
→ 点赞成功 → WebSocket 通知文档作者"您的文档被点赞"
```

### 流程 4：内容管理（管理员）
```
登录 → 进入管理后台 → 选择管理模块（电子书/分类/文档/用户）
→ 执行增删改查操作 → 前端即时反馈
```

### 流程 5：数据统计（管理员）
```
首页查看统计卡片（总阅读/总点赞/今日数据/增长率）
→ 查看 30 天趋势折线图（ECharts）
```

### 流程 6：AI 问答（注册用户）
```
打开 AI 问答面板 → 输入问题 → AI 基于 DeepSeek 知识回答
→ 用户追问 → AI 基于上下文继续回答 → 多轮对话
→ 用量记录到 ai_usage_log
```

### 流程 7：AI 辅助笔记（注册用户）
```
新建笔记 → 输入主题 → 点击"AI 生成" → AI 生成笔记内容
→ 用户编辑修改 → 选择公开/私有 → 保存
```

### 流程 8：AI 辅助文档管理（管理员）
```
编辑文档 → 选中段落 → 点击"AI 辅助" → 选择操作（扩写/润色/补充）
→ AI 生成建议内容 → 管理员确认/修改 → 保存到文档
```

## 6. 数据和工作区模型

### 核心业务对象

| 对象 | 说明 | 所有权 |
|------|------|--------|
| User | 系统用户 | 管理员创建和管理 |
| Category | 分类（支持无限级树形） | 管理员管理 |
| Ebook | 电子书 | 管理员管理，关联两级分类 |
| Doc | 文档（属于某本电子书） | 管理员管理，支持无限级树形 |
| Content | 文档正文（HTML） | 与 Doc 一对一 |
| EbookSnapshot | 电子书每日数据快照 | 系统自动生成 |
| Note | 用户学习笔记 | 用户创建，可公开分享 |
| AiConversation | AI 对话会话 | 用户创建，系统管理 |
| AiMessage | AI 对话消息 | 用户与 AI 的交互记录 |
| AiUsageLog | AI 用量日志 | 系统自动记录 |

### 关键关系
- Ebook → Category（两级：category1_id, category2_id）
- Doc → Ebook（ebook_id）
- Doc → Doc（parent_id，父子树形）
- Content → Doc（一对一）
- EbookSnapshot → Ebook（每日一条）

### 数据隔离
- 无租户模型，单实例部署
- 所有用户共享同一数据空间

## 7. AI / 自动化定位

**AI 功能**（V2 新增）：

- **AI 问答**：用户向 AI 提问海洋生物相关问题，基于 DeepSeek 外部知识回答，支持多轮对话
- **AI 内容生成**：用户可使用 AI 辅助生成学习笔记；管理员可使用 AI 辅助撰写文档
- **用户笔记**：用户可创建学习笔记（富文本），支持公开分享
- **供应商抽象**：可切换 AI 供应商（当前接 DeepSeek，预留通义千问等）

自动化：
- 定时任务：电子书数据快照生成（每分钟）
- 消息队列：点赞通知异步推送（RocketMQ）

## 8. 部署假设

| 项目 | 说明 |
|------|------|
| 部署方式 | 本机 Docker Compose（答辩演示用） |
| 内外部 | 内部演示，非公网访问 |
| 认证方式 | JWT Token，前端存 sessionStorage |
| 租户模型 | 无，单实例 |
| 外部集成 | 无第三方登录、无支付、无邮件 |
| 容器编排 | MySQL + Redis + ElasticSearch + 后端 + Nginx（前端） |

## 9. 核心原则

1. **知识优先**：所有功能围绕知识内容的展示和管理，不做社交、不做商业化
2. **安全底线**：XSS 防御、SQL 注入防御、接口限流、密码加密，缺一不可
3. **操作简单**：管理后台操作不超过 3 步完成，删除需二次确认
4. **前后端分离**：接口契约先行，前端独立部署，后端纯 API 服务

## 10. V1 方向

**原型级交付**，核心目标：

1. 完整的知识浏览链路：首页 → 电子书列表 → 文档阅读 → 搜索
2. 完整的内容管理链路：电子书/分类/文档/用户的增删改查
3. 基本互动功能：点赞（Redis 防重）+ WebSocket 通知
4. 数据统计：快照任务 + 首页统计卡片 + ECharts 趋势图
5. 安全防护：JWT 认证、XSS 过滤、接口限流、密码加密
6. 全文检索：ElasticSearch 基础搜索 + 高亮
7. 可部署：Docker Compose 一键启动

## 11. 非目标（V1 明确不做）

- 用户自主注册（由管理员在后台添加）
- 第三方登录（微信/GitHub/Gitee）
- 评论/收藏/阅读历史
- 权限细化（RBAC），仅管理员/普通用户两档
- 内容版本控制/回滚
- 数据导出（PDF/Excel）
- 移动端适配
- CI/CD 流水线
- 微服务拆分
- AI 流式输出（SSE），V1 使用同步请求

## 13. 开放问题

| 问题 | 影响范围 | 建议 |
|------|---------|------|
| 富文本内容是否需要支持特殊标签（视频、音频）？ | 文档编辑器配置 | V1 仅支持图文混排，视频/音频 V2 再考虑 |
| 分类层级是否需要限制？ | 数据模型设计 | 不限制，但电子书只挂到二级分类 |
| 快照数据保留多久？ | 存储空间 | V1 保留 30 天，超期清理 |
