# 海洋生物知识库 PRD

## 1. 文档目的

本 PRD 定义海洋生物知识库系统的功能需求、权限规则、验收标准和交付边界。后续产物（数据模型、API 契约、安全设计）均以本文档为依据。

## 2. 产品目标

### 业务目标
- 构建一个集中化的海洋生物知识浏览平台
- 支持管理员高效管理知识内容（电子书、分类、文档）
- 通过数据统计展示知识库的使用情况和热度

### 用户目标
- 普通用户：快速找到感兴趣的海洋生物知识，阅读体验良好
- 注册用户：能参与互动（点赞），获得反馈（通知）
- 管理员：操作简单，内容管理效率高

### 非目标
- 不做社交功能（评论、关注、私信）
- 不做内容付费
- 不做移动端

## 3. 用户和角色

## 5. 核心用户旅程

### 旅程 1：知识浏览

- **Actor**：普通用户
- **前置条件**：无
- **步骤**：
  1. 访问首页，查看统计概览和电子书列表
  2. 按分类筛选或搜索关键词
  3. 点击电子书进入文档阅读页
  4. 左侧目录树选择文档，右侧显示富文本内容
- **成功结果**：用户完成阅读，阅读数 +1
- **失败状态**：电子书/文档不存在时显示空状态提示

### 旅程 2：文档点赞

- **Actor**：注册用户
- **前置条件**：已登录
- **步骤**：
  1. 阅读文档时，点击点赞按钮
  2. 后端校验：同 IP 同文档每天只能点赞一次（Redis 控制）
  3. 点赞成功，文档点赞数 +1
  4. 通过 WebSocket 通知文档所属电子书的关注者
- **成功结果**：点赞数更新，通知推送
- **失败状态**：今日已点赞，提示"今日已点赞"

### 旅程 3：内容管理

- **Actor**：管理员
- **前置条件**：已登录，拥有管理员权限
- **步骤**：
  1. 进入管理后台
  2. 选择管理模块（电子书/分类/文档/用户）
  3. 执行增删改查操作
  4. 操作完成后前端即时刷新列表
- **成功结果**：数据变更成功，前端提示"操作成功"
- **失败状态**：参数校验失败返回中文提示，删除需二次确认

### 旅程 4：用户登录

- **Actor**：注册用户/管理员
- **前置条件**：已有账号
- **步骤**：
  1. 输入登录名和密码
  2. 后端验证：登录名存在 + 密码 MD5+盐值匹配
  3. 生成 JWT Token，存入 Redis
  4. 返回 Token 给前端，存入 Vuex + sessionStorage
- **成功结果**：登录成功，跳转首页，导航栏显示用户信息
- **失败状态**：登录名或密码错误，提示"登录名或密码错误"

## 6. 生命周期和状态机

### 电子书
- **状态**：无显式状态，创建即存在，删除即消失
- **操作**：创建 → 编辑 → 删除（硬删除）
- **审计**：记录 doc_count、view_count、vote_count

### 文档
- **状态**：无显式状态
- **操作**：创建 → 编辑 → 删除（含递归删除子文档）
- **审计**：记录 view_count、vote_count

### 用户
- **状态**：无显式状态（无禁用/启用机制，V1 简化）
- **操作**：创建 → 编辑 → 删除
- **敏感数据**：密码不返回前端，日志不打印密码

### 点赞
- **幂等规则**：同 IP + 同文档 + 同一天 = 唯一
- **存储**：Redis Key `vote:{docId}:{ip}:{date}`，TTL 24h

- 跨域配置
- 前端路由守卫（未登录不可访问管理页）

- 前端错误页面（404/500）

## 9. 模块需求

### 9.1 电子书管理

- **目的**：管理知识库的顶层组织单元
- **角色**：管理员（CRUD）、所有用户（查看）
- **关键对象**：Ebook（id, name, description, cover, category1_id, category2_id, doc_count, view_count, vote_count）
- **主要动作**：
  - 列表查询：支持分页、名称模糊搜索、分类过滤
  - 新增：封面上传、名称必填、分类必选，雪花算法生成 ID
  - 编辑：同新增规则
  - 删除：二次确认，删除后关联文档一并处理
- **校验规则**：
  - 名称：非空，最大 50 字符
  - 描述：最大 200 字符
  - 分类：必须选择
- **验收标准**：
  - Given 管理员在电子书管理页 When 填写信息并提交 Then 电子书创建成功，列表刷新显示
  - Given 管理员点击删除 When 确认删除 Then 电子书及关联数据消失

### 9.2 分类管理

- **目的**：为电子书提供多级分类体系
- **角色**：管理员（CRUD）、所有用户（查看）
- **关键对象**：Category（id, parent, name, sort）
- **主要动作**：
  - 树形展示：按 sort 升序，以树形表格显示父子关系
  - 新增/编辑：父分类通过下拉树选择，不能将自己设为父分类
  - 删除：若存在子分类，禁止删除并提示
- **校验规则**：
  - 名称：非空，最大 20 字符
  - 父分类：不能是自己或自己的后代
- **验收标准**：
  - Given 管理员新增分类 When 选择父分类并提交 Then 分类树刷新显示新节点
  - Given 分类下有子分类 When 点击删除 Then 提示"该分类下存在子分类，无法删除"

### 9.3 文档管理

- **目的**：管理电子书下的具体内容
- **角色**：管理员（CRUD）、所有用户（阅读）
- **关键对象**：Doc（id, ebook_id, parent, name, sort, view_count, vote_count）、Content（id, content）
- **主要动作**：
  - 树形展示：某个电子书下的所有文档，支持无限级
  - 新增/编辑：名称、父节点、排序、富文本内容一并保存（doc 表 + content 表）
  - 删除：递归删除所有子文档及对应 content
  - 阅读：目录树 + 富文本渲染，阅读时 view_count +1
- **校验规则**：
  - 名称：非空，最大 50 字符
  - 内容：HTML 格式，需 XSS 过滤
  - 排序：整数
- **验收标准**：
  - Given 管理员编辑文档 When 保存 Then doc 和 content 同步更新
  - Given 用户阅读文档 When 页面加载 Then 阅读数 +1，内容正确渲染

### 9.4 用户管理

- **目的**：维护系统用户账号
- **角色**：管理员（CRUD）
- **关键对象**：User（id, login_name, name, password）
- **主要动作**：
  - 新增：登录名唯一，密码 MD5+盐值加密
  - 编辑：可改昵称，登录名不可改，密码不显示
  - 删除：二次确认
  - 重置密码：密码需符合复杂度（数字+字母，6-32 位）
- **校验规则**：
  - 登录名：非空，唯一，最大 50 字符
  - 昵称：非空，最大 50 字符
  - 密码：6-32 位，必须包含数字和字母
- **验收标准**：
  - Given 管理员新增用户 When 登录名重复 Then 提示"登录名已存在"
  - Given 管理员重置密码 When 密码不符合规则 Then 提示密码复杂度要求

### 9.5 登录认证

- **目的**：验证用户身份，保护系统安全
- **角色**：所有已注册用户
- **关键对象**：JWT Token
- **主要动作**：
  - 登录：验证登录名+密码，生成 JWT，存入 Redis，返回前端
  - 退出：清除 Redis 中的 Token
  - 拦截：除登录和部分公共接口外，其他接口需携带有效 Token
  - 登录状态持久化（UR-07）：前端使用 Vuex + sessionStorage 保存登录信息，刷新页面保持登录
  - 前端路由拦截（UR-09）：未登录用户无法访问管理页面，自动跳转首页
  - 动态菜单展示：导航栏和侧边菜单根据登录状态和角色动态显示/隐藏管理入口
- **校验规则**：
  - Token 过期时间：24 小时
  - 401 响应：Token 无效或过期
- **验收标准**：
  - Given 用户登录 When 凭据正确 Then 返回 Token，跳转首页，导航栏显示用户信息
  - Given Token 过期 When 访问接口 Then 返回 401，前端跳转登录页
  - Given 用户刷新页面 When sessionStorage 有 Token Then 保持登录状态，不丢失
  - Given 未登录用户 When 访问 /admin 路由 Then 自动跳转首页
  - Given 已登录用户 When 查看导航栏 Then 管理员可见管理入口，普通用户不可见

### 9.6 数据统计

- **目的**：展示知识库使用情况和热度趋势
- **角色**：所有用户（查看）
- **关键对象**：EbookSnapshot（ebook_id, date, view_count, vote_count, view_increase, vote_increase）
- **主要动作**：
  - 快照生成：定时任务（每分钟），记录每个电子书的总阅读/点赞及当日增量
  - 首页统计：总阅读量、总点赞量、点赞率、今日阅读/点赞、预计今日阅读、环比增长率
  - 趋势图：近 30 天阅读增长和点赞增长折线图
- **幂等保证**：INSERT + UPDATE 或 ON DUPLICATE KEY UPDATE
- **验收标准**：
  - Given 快照任务执行 When 同一天重复执行 Then 数据不重复不遗漏
  - Given 用户访问首页 When 页面加载 Then 统计卡片和趋势图数据正确

### 9.7 全文检索

- **目的**：支持文档内容的高效搜索
- **角色**：所有用户
- **关键对象**：ElasticSearch 索引
- **主要动作**：
  - 索引同步：文档保存/更新时同步写入 ES
  - 搜索：关键词匹配，支持高亮和分页
- **验收标准**：
  - Given 用户搜索关键词 When 存在匹配文档 Then 返回高亮结果列表
  - Given 管理员保存文档 When 保存成功 Then ES 索引同步更新

### 9.8 实时通知

- **目的**：提升用户互动体验
- **角色**：注册用户
- **关键对象**：WebSocket Session
- **主要动作**：
  - 连接：用户登录后建立 WebSocket 连接
  - 通知：文档被点赞时，通过 RocketMQ 异步推送给在线用户
- **验收标准**：
  - Given 用户 A 在线 When 用户 B 点赞 A 的文档 Then A 收到通知弹窗

## 10. AI / 自动化需求

本系统不涉及 AI 功能。

自动化需求：
- 定时任务：EbookSnapshotJob，每分钟执行，生成电子书数据快照
- 消息队列：RocketMQ 消费点赞事件，异步发送 WebSocket 通知

## 12. 非功能需求

### 12.1 性能要求

| 指标 | 要求 |
|------|------|
| 接口响应时间 | 95% 的接口请求在 500ms 内返回（不含富文本内容上传下载） |
| 并发支持 | 至少 100 个并发用户同时访问，主要查询接口通过分页、缓存优化 |
| 定时任务效率 | 电子书快照生成每次执行耗时不超过 5 秒（数据量 1 万级） |

### 12.2 安全性要求

| 维度 | 要求 |
|------|------|
| 密码存储 | MD5 + 固定盐值，不返回前端，日志不打印 |
| XSS 防御 | 所有用户输入 HTML 转义，富文本白名单过滤 |
| SQL 注入 | MyBatis-Plus 参数化查询，禁止拼接 |
| 接口限流 | 登录接口（防暴力破解）、点赞接口（防刷） |
| Token 安全 | JWT + Redis 双重校验，过期自动失效 |
| 敏感信息 | 响应中不返回密码字段，日志中不打印 Token/密码 |
| 跨域 | 配置 CORS 白名单 |

### 12.3 可用性与易用性

- **界面风格**：Ant Design Vue 组件库，统一、美观、响应式
- **操作反馈**：所有增删改操作明确成功/失败提示，删除操作二次确认
- **错误提示**：业务错误返回可读中文描述，不暴露技术细节
- **文档结构**：树形目录，富文本阅读体验良好

### 12.4 可维护性

- **代码规范**：后端 Lombok + 包命名规范（controller/service/mapper），前端 TypeScript + ESLint
- **日志追踪**：MDC 自动生成唯一 LOG_ID
- **配置分离**：多环境配置（dev/prod），通过环境变量切换
- **模块化**：电子书、分类、文档、用户等模块低耦合

### 12.5 可扩展性

- **消息队列**：预留 RocketMQ，未来可扩展邮件通知、数据同步等异步任务
- **微服务化**：现有模块可独立拆分为微服务
- **第三方登录**：可扩展 OAuth2.0（微信、GitHub）
- **全文检索**：可接入 ElasticSearch 实现高效搜索

## 13. 验收标准（汇总）

### 核心流程验收

| 场景 | Given | When | Then |
|------|-------|------|------|
| 浏览知识 | 用户在首页 | 点击电子书 → 选择文档 | 富文本内容正确渲染，阅读数 +1 |
| 搜索文档 | 用户在搜索框 | 输入关键词并搜索 | 返回匹配结果，关键词高亮 |
| 点赞文档 | 已登录用户在阅读页 | 点击点赞按钮 | 点赞数 +1，触发通知 |
| 防重复点赞 | 同 IP 已点赞 | 再次点击点赞 | 提示"今日已点赞" |
| 管理电子书 | 管理员在管理页 | 新增/编辑/删除电子书 | 操作成功，列表刷新 |
| 管理文档 | 管理员在管理页 | 编辑文档并保存 | doc 和 content 同步更新 |
| 用户登录 | 用户在登录页 | 输入正确凭据 | 登录成功，跳转首页 |
| Token 过期 | Token 已过期 | 访问任意接口 | 返回 401，跳转登录页 |
| 路由守卫 | 未登录用户 | 访问管理页 URL | 自动跳转首页 |
| 数据统计 | 用户在首页 | 查看统计区域 | 卡片数据正确，趋势图加载 |

## 14. 开放问题

| 问题 | 影响 | 建议决策 |
|------|------|---------|
| 电子书封面存储方式 | 文件上传模块设计 | 本地文件系统（Docker Volume 挂载） |
| 文档内容大小限制 | 数据库性能 | 单篇文档 HTML 内容不超过 1MB |
| WebSocket 连接策略 | 通知可靠性 | 页面加载时连接，断线不自动重连（V1 简化） |
