## 二、技术实现要点

### 数据库表设计

#### 竞赛题目表
```sql
CREATE TABLE `question` (
    `id` BIGINT NOT NULL COMMENT '题目ID',
    `quiz_id` BIGINT NOT NULL COMMENT '所属竞赛ID',
    `category` VARCHAR(20) NOT NULL COMMENT '分类：science/biology/environment',
    `type` VARCHAR(20) NOT NULL COMMENT '题型：single/multiple/true_false',
    `difficulty` INT NOT NULL DEFAULT 1 COMMENT '难度：1-5',
    `content` TEXT NOT NULL COMMENT '题目内容',
    `answer` VARCHAR(500) NOT NULL COMMENT '正确答案',
    `analysis` TEXT COMMENT '答案解析',
    `sort` INT NOT NULL DEFAULT 0 COMMENT '排序',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_quiz_id` (`quiz_id`),
    KEY `idx_category` (`category`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='竞赛题目表';
```

#### 答题记录表
```sql
CREATE TABLE `quiz_record` (
    `id` BIGINT NOT NULL COMMENT '记录ID',
    `user_id` BIGINT NOT NULL COMMENT '用户ID',
    `quiz_id` BIGINT NOT NULL COMMENT '竞赛ID',
    `score` INT NOT NULL DEFAULT 0 COMMENT '得分',
    `total_score` INT NOT NULL COMMENT '总分',
    `correct_count` INT NOT NULL DEFAULT 0 COMMENT '正确题数',
    `total_count` INT NOT NULL COMMENT '总题数',
    `start_time` DATETIME COMMENT '开始时间',
    `end_time` DATETIME COMMENT '结束时间',
    `status` VARCHAR(20) NOT NULL DEFAULT 'in_progress' COMMENT '状态：in_progress/completed',
    `create_time` DATETIME DEFAULT CURRENT_TIMESTAMP,
    `update_time` DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    PRIMARY KEY (`id`),
    KEY `idx_user_id` (`user_id`),
    KEY `idx_quiz_id` (`quiz_id`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='答题记录表';
```

### 题目批量导入
```java
@Transactional
public void batchImportQuestions(List<Question> questions) {
    // 分批插入，每批100条
    int batchSize = 100;
    for (int i = 0; i < questions.size(); i += batchSize) {
        int end = Math.min(i + batchSize, questions.size());
        List<Question> batch = questions.subList(i, end);
        batch.forEach(question -> questionMapper.insert(question));
    }
}
```

## 五、遇到的问题与解决方案

**问题1**：多选题答案存储格式不统一  
**解决**：使用逗号分隔存储，如"A,B,C"，便于后续比对答案

**问题2**：题目解析内容过长影响查询性能  
**解决**：查询题目列表时不加载解析字段，仅在查看详情时加载

**问题3**：批量导入时事务过大  
**解决**：分批插入，每批100条，避免长事务
