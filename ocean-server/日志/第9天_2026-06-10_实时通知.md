## 二、技术实现要点

### WebSocket服务端
```java
@ServerEndpoint("/ws/{userId}")
@Component
public class WebSocketServer {
    
    // 存储所有在线用户的会话
    private static ConcurrentHashMap<Long, Session> sessionMap = new ConcurrentHashMap<>();
    
    @OnOpen
    public void onOpen(Session session, @PathParam("userId") Long userId) {
        sessionMap.put(userId, session);
        broadcastOnlineCount();
    }
    
    @OnClose
    public void onClose(@PathParam("userId") Long userId) {
        sessionMap.remove(userId);
        broadcastOnlineCount();
    }
    
    @OnMessage
    public void onMessage(String message, @PathParam("userId") Long userId) {
        log.info("收到用户{}的消息: {}", userId, message);
    }
    
    // 向指定用户推送消息
    public static void sendMessage(Long userId, String message) {
        Session session = sessionMap.get(userId);
        if (session != null && session.isOpen()) {
            session.getAsyncRemote().sendText(message);
        }
    }
    
    // 广播在线用户数
    private void broadcastOnlineCount() {
        String msg = "{\"type\":\"online_count\",\"count\":" + sessionMap.size() + "}";
        sessionMap.values().forEach(session -> {
            session.getAsyncRemote().sendText(msg);
        });
    }
}
```

## 五、遇到的问题与解决方案

**问题1**：WebSocket连接需匹配后端端口与token参数  
**解决**：连接地址使用ws://localhost:8080/ws?token=xxx，携带JWT token认证

**问题2**：多服务器部署时WebSocket会话不共享  
**解决**：使用Redis存储会话信息，实现分布式WebSocket（后续优化）

**问题3**：连接断开后未清理资源  
**解决**：在@OnClose回调中清理sessionMap，防止内存泄漏
